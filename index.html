Setting Up

Installing things we'll need later

There are several tools I'm going to rely on during this presentation. About half of them are tools to install the other tools. If you have these already, don't just rest on your laurels (if you even remembered to bring your laurels) - find someone who's installing things and offer a hand. If you are seeing these tools for the first time tonight, you're in for a treat - these goodies are going to save you hours, and let you do some really cool stuff!

Homebrew

Because Apple didn't give us this functionality

We'll start with Homebrew, a thing to install other things.

More precisely, Homebrew is a package manager for OSX, which is used to install Unix tools and other open source software packages (though in conjunction with the next tool it can help with applications as well).

To install Homebrew, open the terminal and type (or copy)

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

You can confirm that it installed properly by typing

brew -v

Homebrew Cask

Drag this

Homebrew Cask is an installation manager for OSX. It makes installing applications fun! And by "fun" I mean "much less of a headache".

brew install caskroom/cask/brew-cask

Confirm

brew cask --version

Git

'er done

In general, using Git is right between flossing and wearing a seat belt on the list of Things You Should Be Doing (see here for a quick rundown of why if you're not familiar with Git or version control). While we won't be using Git directly, some of the tools I'll describe use it under the hood, so we need to have it installed. brew install git

Confirm

git -v

Node.js

"Why not put JavaScript on a server?" is apparently a question someone asked once

Node.js is a runtime environment for developing server-side web applications in JavaScript. Like everything connected to JavaScript, it has grown faster and farther from its original purposes than Mark Ruffalo in a gamma ray machine, and JavaScript developers have used Node.js to power all sorts of tools, including packages that will watch for changes in Sass files, convert the Sass into CSS, manage front end code libraries, and order sandwiches, all from the command line. Node.js has a modular architecture, and to assist in managing them all there is one module called node package manager (npm) which is installed as part of Node.js. Node/npm are a pretty big deal; npmjs.com lists 196,271 packages and over two billion downloads in the past month.

Installing Node.js

Node.js can be installed with Homebrew:

brew install node

Confirm that node (and npm) were installed properly with

node -v
npm -v

Sass

Everything CSS should be

Of all the pieces of software I'm asking you to install, Sass is the most important for this course. Sass was originally written in the lovely and beguiling language of Ruby. Unfortunately, while Ruby is fast (and delightful) to work in, it runs pretty slow. Fortunately, Sass has been rewritten in C++, and even better there's a wrapper for the C++ version available as a node package.

Cloud 9 Users Start Here

This is the only bit of the installation process that you need. Everything else is either already installed, or doesn't make sense to use with Cloud 9.

node-sass

We're going to use node-sass to invoke Sass from the command line. At first we'll type out a command every time we want that to happen, then we'll get frustrated with the tedium and write a file and download a bunch more things so we never have to do it manually again.

We're developers. That's what we do.

npm install -g node-sass

To confirm that you have node-sass installed, without being bothered about all the dependencies that were installed with it, type

npm ls --depth=0
