Crash Sass

That's Hard to Say.

Your First Sass File

The simplest thing

Make a new directory to hold your Sass files.

mkdir scss

Now move the CSS file this page is using, css/blog.css, into that directory.

mv css/blog.css scss/blog.scss

If you refresh the blog page in the browser, it will look...not great. There's some residual Bootstrap styling, but not much. Let's fix that. Regenerate the blog.css file using node-sass:

node-sass scss/blog.scss css/blog.scss

Now refresh the blog page again, and behold! We're back where we started!

Why That Matters

The point here is that Sass is a superset of CSS. That is, any valid CSS file (like the original blog.css) is also a valid Sass file (like blog.scss). That makes it easy to convert a project with existing CSS files into Sass, because they already fit in the Sass syntax.

Also, it's always a good idea to make sure simple things work before you try the real magic.

Indentation

It's so much neater this way

If you look at this (or any other) web page in the Developer Tools > Elements tab, you'll see a nice tree structure - divs inside the body, h2's and p's inside the divs, etc. But look at the blog.css file - at the "blog posts" section, around line 136. Repetition! Selectors written over and over again! I can't stand it!

HTML has a natural tree structure, and CSS should too. It doesn't - but Sass does. Open scss/blog.scss and rewrite the section with the repeated selectors like so:

Before Indentation


.blog-post {
  margin-bottom: 60px; }

.blog-post h2 {
  margin-bottom: 5px;
  font-size: 40px; }

.blog-post p.meta {
  margin-bottom: 20px;
  color: #999; }

After Indentation


.blog-post {
  margin-bottom: 60px;

  h2 {
    margin-bottom: 5px;
    font-size: 40px; }

  p.meta {
    margin-bottom: 20px;
    color: #999; }
}

Now refresh the browser again, and voila! Nothing has changed!

No, I mean literally nothing has changed. We edited the Sass file, but the HTML file isn't connected to Sass, it's connected to the CSS file. Regenerate that one again

node-sass scss/blog.scss css/blog.css

And there we go!

Math

No, I mean make the computer do math

There's a strange number in the CSS file:

line-height: 1.42857143;

It actually has a perfectly good reason for being there: The font is 14px, and with a line-height of 1.42857143, the line-height ends up with a nice round value of 20px, because 14 * 1.42857143 = 20 (almost exactly). But what if that font size changes? What are you supposed to do, open up a calculator utility or something? What are we, Neanderthals?

Division? That's Sassy!

Open scss/blog.scss and replace that line-height directive with something a bit clearer:

line-height: (20 / 14);
